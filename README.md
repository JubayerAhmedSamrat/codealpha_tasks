# CodeAlpha Tasks

A comprehensive collection of educational projects demonstrating core computer science concepts and practical software development skills.

## Project Overview

This repository contains four distinct projects, each designed to illustrate fundamental programming principles and algorithmic problem-solving techniques.

---

## Project 1: CGPA Calculator

### Description
A high-performance console application designed to automate the calculation of academic performance metrics and cumulative GPA.

### Key Features
- **Dynamic Input Management**: Users can specify the exact number of courses taken during the semester
- **Detailed Grade Processing**: Captures individual grades and credit hours for each subject
- **Automated Calculations**:
  - Computes total credits earned
  - Calculates total grade points using: `Total Grade Points = Σ(Grade × Credit Hours)`
- **Comprehensive Reporting**: Displays a detailed breakdown of individual course grades, semester GPA, and cumulative CGPA

## Project 2: Login and Registration System

### Description
A secure credential management system designed to handle user authentication through persistent file storage and validation protocols.

### Key Features
- **User Registration**: Dedicated functionality to accept and validate new usernames and passwords
- **Security & Validation**: Implements duplicate username detection to ensure identity uniqueness
- **Persistent Storage**: Securely stores user credentials in local file or database format
- **Authentication Logic**: Login verification that validates user credentials against stored data
- **User Feedback**: Clear success and error messaging for all authentication attempts

---

## Project 3: Sudoku Solver

### Description
An advanced algorithmic implementation utilizing the Backtracking algorithm to solve complex 9×9 Sudoku puzzles efficiently.

### Key Features
- **Grid Representation**: Sudoku board managed and manipulated as a 2D array structure
- **Constraint Validation**: Enforces three critical constraints before placing any number:
  - Row uniqueness
  - Column uniqueness
  - 3×3 subgrid uniqueness
- **Backtracking Engine**: Recursively explores possible number placements and backtracks when reaching a dead end until the puzzle is solved

---

## Project 4: Banking System

### Description
A scalable banking application built using Object-Oriented Programming (OOP) principles to manage financial operations and customer data.

### Key Features
- **Class-Based Architecture**: Robust implementation of Customer, Account, and Transaction classes with specific attributes
- **Account Management**: Tools to create new customer profiles and manage existing bank accounts
- **Financial Operations**:
  - Secure deposits and withdrawals
  - Internal fund transfers between accounts
- **Transaction Reporting**:
  - Maintains detailed transaction history logs
  - Real-time balance viewing and recent activity details
